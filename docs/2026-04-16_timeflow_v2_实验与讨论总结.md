## 2. 模型与推理相关结论

### 2.1 为什么新模型推理时不需要重新依赖训练集

新模型的 checkpoint：

- `scripts/timeflow_v2/outputs/best_flow_v2.pt`

在保存时不仅存了模型权重，还存了：

- `label_mean`
- `label_std`
- `num_channels`
- `param_dim`

因此新模型推理时：

- 仍然需要验证集样本作为输入；
- 但不需要再去训练集目录重新统计标签均值和标准差。

而旧模型的 checkpoint：

- `scripts/timeflow_v2/outputs/best_flow_model.pth`

本质上只是 `state_dict`，没有保存训练时的 `label_mean/std`，所以推理时若想把标准化后的标签恢复到真实物理尺度，就需要：

- 重新从训练集目录统计；
- 若训练集目录不存在，则退回到验证集目录统计。

---

### 2.2 `log_prob` 的含义

`log_prob(theta_true | x)` 表示：

- 给定输入样本 `x`；
- 模型对真实参数 `theta_true` 赋予的**对数概率密度**。

其与训练时的 `NLL` 的关系是：

```text
NLL = -log_prob
```

因此：

- `log_prob` 越大越好；
- `NLL` 越小越好。

## 3. 单样本对比实验结果

在同一个验证样本上，新旧模型的对比结果为：

```text
New model log_prob: 16.041225
Old model log_prob: -7.395089
New model MAE: 2.147265
Old model MAE: 1.853954
```

### 3.1 结果解读

这说明：

- **新模型在该样本上的联合概率建模明显更好**；
- **旧模型在该样本上的后验均值点估计更接近真实值**。

也就是说：

- 新模型更擅长学习 `p(theta | x)`；
- 旧模型在这个单样本上的均值预测更“贴近真值”。

因此对于单样本来说，两者可能会出现：

- 新模型 `log_prob` 更好；
- 老模型 `MAE` 略好；

这并不矛盾，因为：

- `log_prob` 衡量的是概率建模；
- `MAE` 衡量的是点估计误差。

## 4. 多验证集样本平均结果

在多个验证集样本上的平均结果为：

```text
Average new model log_prob: 15.723892
Average old model log_prob: -7.294945
Average new model MAE: 3.469410
Average old model MAE: 3.578722
```

### 4.1 结果解读

这个结果说明：

1. **新模型在联合后验概率建模上显著优于旧模型**
   - 新模型平均 `log_prob` 远高于旧模型；
   - 说明真实标签在新模型后验分布中落入高概率区域的程度更高。

2. **新模型在点估计误差上也略优于旧模型**
   - 新模型平均 `MAE` 比旧模型更小；
   - 虽然差距不如 `log_prob` 明显，但方向一致。

### 4.2 总结性结论

可以归纳为：

> 新模型不仅在联合后验分布建模上明显优于旧模型，而且在后验均值的点估计误差上也略优于旧模型，说明 `TimeFlow V2` 在整体性能上优于旧模型。

如果从研究目标角度看，这一点尤其重要，因为本课题更关注：

- `p(theta | x)` 的学习能力；
- 而不仅仅是单点回归精度。

因此：

> `log_prob` 的显著提升比 `MAE` 的小幅提升更具有研究意义。

## 5. 图像可视化讨论

### 5.1 图中的标记含义

后验图中的标记含义如下：

- **红色竖线**：真实标签值；
- **黑色虚线**：后验样本均值；
- **红色 x**：真实标签在二维参数平面中的位置。

### 5.2 为什么原来的二维图看起来像很多模糊小方格

因为非对角线部分最初采用的是：

```python
ax.hist2d(...)
```

即二维直方图热力图。  
这种画法本质上是把采样点分到很多小网格里，再根据每个格子的计数显示颜色深浅，因此会呈现为：

- 一些模糊的小方形块；
- 更像像素格，而不是平滑曲线。

### 5.3 后来的改进

后续已将新旧模型的后验二维分布绘图方式改为：

- `contourf(...)`
- `contour(...)`

也就是：

- 填充等高线图；
- 配合轮廓线边界；

这样更接近论文中常见的联合后验分布展示风格。

同时，原始的热力图版本也已单独保留为备份脚本。

## 7. 训练集标签统计缓存

由于旧模型 checkpoint 不保存 `label_mean/std`，推理时需要重新统计训练集标签分布。  
为了避免每次都重新扫描训练集，后续加入了缓存机制：

- 缓存文件：
  - `scripts/timeflow_v2/result_fig/old_model_label_stats_cache.npz`

作用：

- 首次运行时扫描训练集并保存缓存；
- 后续直接读取缓存，加速推理与对比脚本运行。

同时，对于耗时步骤也补充了进度提示，例如：

- `Collecting old-model label stats`
- `Sampling posterior...`
